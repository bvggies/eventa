# Contributing to Eventa

Thank you for your interest in contributing to Eventa!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Set up the development environment (see [SETUP.md](./SETUP.md))
4. Create a new branch for your feature
5. Make your changes
6. Test thoroughly
7. Submit a pull request

## Code Style

- Use TypeScript for type safety
- Follow existing code patterns
- Write clear, descriptive commit messages
- Add comments for complex logic

## Testing

- Test on both iOS and Android (for mobile)
- Test all user flows
- Verify API endpoints work correctly
- Check for console errors

## Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass
3. Request review from maintainers
4. Address any feedback

## Questions?

Open an issue or contact the development team.
